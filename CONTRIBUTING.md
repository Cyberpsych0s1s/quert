# Contributing to Quert

Contributions are welcome. Bug fixes, features, docs, tests, cleanup, whatever you've got. This is a learning-friendly project, so don't stress about getting everything perfect. Open the PR and we'll sort it out together. There's one real formality: by submitting a contribution you agree it's licensed under the project's [Apache 2.0 license](LICENSE), the same one covering the rest of the code, and that it's your own work or something you otherwise have the right to contribute.

Getting started is the usual dance. Fork and clone (git clone https://github.com/cyberpsych0s1s/quert.git), check that you're on a recent Go (go.mod has the exact version), and branch off main. Run make fmt, make build, and make test before you push so CI stays green; make help lists the rest. Keep the commit message short. Conventional Commits like fix: / feat: / docs: are nice but not required. Then open a pull request. If you get stuck, open an issue or just ask in the PR.
